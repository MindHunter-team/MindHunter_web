# AI学术审查系统 - AI 学术论文评估系统使用说明

## 1. 环境准备

### Python 版本
- Python 3.10 或更高版本

### 安装依赖

```bash
cd 完整后端
pip install -r requirements.txt
```

---

## 2. 配置 API Key（千问 QWEN）

### 获取 API Key
1. 访问 [阿里云百炼控制台](https://dashscope.console.aliyun.com/)
2. 注册/登录阿里云账号
3. 开通"百炼"大模型服务
4. 在"API-KEY 管理"中创建一个新的 Key（格式：`sk-xxxx`）

### 写入配置文件

打开 `evaluation_agents/.env` 文件，将占位符替换为你的真实 Key：

```
# 修改前
DASHSCOPE_API_KEY=sk-your-api-key-here

# 修改后（示例）
DASHSCOPE_API_KEY=sk-ws-xxxxxxxxxxxxxxxx
```

> 注意：`.env` 文件不要截图、不要上传到公开仓库。

---

## 3. 运行系统

### 基本命令

```bash
python main_controller/main.py "你的论文.pdf"
```

### 输出评估报告到文件

```bash
python main_controller/main.py "你的论文.pdf" -o result.json
```

### 自定义重试次数（默认 3 次）

```bash
python main_controller/main.py "你的论文.pdf" -o result.json --max-retries 2
```

---

## 4. 输出说明

运行完成后生成的 `result.json` 包含：

| 字段 | 含义 |
|------|------|
| `paper_data` | 论文解析结果（标题、作者、期刊、引用量、全文） |
| `final_results` | 4 个 Agent 的最终评估（data_reliability / ethics_bias / logical_rigor / innovation） |
| `audit_log` | 审计日志（幻觉校验 + 跨引擎冲突仲裁记录） |
| `retry_count` | 实际重试次数 |
| `audit_passed` | 审计是否通过（true/false） |

## 6. 常见问题

### Q: 报错 `DASHSCOPE_API_KEY not found`
确认 `evaluation_agents/.env` 文件存在且 Key 已正确填入。

### Q: 报错 `ModuleNotFoundError: No module named 'openai'`
执行 `pip install -r requirements.txt`。

### Q: Semantic Scholar 查询无结果
正常现象。该 API 为免费公开接口，部分中文论文可能检索不到，系统会自动 Fallback。

### Q: 运行很慢
正常。系统需要调用 LLM 5~6 次（元数据提取 + 4个Agent + 审计），每次约 10~30 秒。
